Hybrid Dual-Track Query System Design

Simple queries: pure freeform. Complex queries: freeform + optional builder fallback.

SIMPLE QUERIES (Keep as-is)
User types → Smart parser → Direct answer
Examples: "How many times has Grace scored 50+?", "What's Liz's highest goal assists?"
No builder needed; parse directly to results
COMPLEX QUERIES (New)
User types → Attempt smart parse → Success? Yes → Direct answer
↓ No
Show error + suggestion
Offer "Use builder" button
↓ (User clicks)
Multi-step builder opens
Pre-populates parsed hints
User refines → Submit
FOUR NEW QUERY SHAPES
1. Comparison: "Vixens vs Swifts, goal assists in 2025"
2. Complex Combination: "Players with 40+ goals AND 5+ gains in 2024"
3. Multi-Season Trend: "Grace Nweke goal assists across 2023, 2024, 2025"
4. All-Time Record: "Highest single-game intercepts, all time + ranking"

Design principle: Users trying simple queries never see the builder. Users attempting complex queries get a clear error + escape hatch. Power users can compose anything step-by-step.
